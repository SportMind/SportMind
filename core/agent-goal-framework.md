# Agent Goal Framework

**The framework for SportMind agents that set and pursue intermediate goals
toward longer-horizon objectives — moving from plan-executors to plan-developers.**

The autonomous agent framework (`core/autonomous-agent-framework.md`) defines
how agents execute pre-defined plans: initialise, monitor, analyse, act, escalate.
This document extends that model with goal-setting — the ability for an agent to
identify what it should be working toward next, set intermediate milestones, evaluate
whether those milestones are being achieved, and adjust its strategy based on what it
observes.

This is not AGI. It is structured goal-directed behaviour within the sports intelligence
domain — a meaningful step toward agents that are genuinely useful over long time horizons
rather than just reliable executors of static plans.

---

## The goal hierarchy model

SportMind agents operate across three goal levels:

```
LEVEL 1 — TERMINAL GOAL (set at deployment; never changes):
  The ultimate purpose of this agent's existence.
  Examples:
    "Maximise the commercial value of $PSG fan token for this portfolio"
    "Provide pre-match intelligence for all UCL matches this season"
    "Track player commercial trajectories for 20 athletes on behalf of this agency"
  
  Terminal goals are set by the human deploying the agent.
  They define success. They do not change unless the deployment is reconfigured.

LEVEL 2 — INSTRUMENTAL GOALS (set by agent; changes as conditions evolve):
  What the agent must achieve to make progress toward the terminal goal.
  Examples:
    "Ensure macro state is current before every market open"
    "Confirm squad selection for all PSG matches in the next 30 days"
    "Complete baseline APS assessment for all 20 tracked athletes"
  
  Instrumental goals are set by the agent based on what it observes.
  They are revised when conditions change, when they are achieved, or
  when they prove irrelevant to the terminal goal.

LEVEL 3 — IMMEDIATE TASKS (generated by the agent each cycle):
  The specific actions needed to progress toward instrumental goals.
  Examples:
    "Refresh macro state (8h old, approaching stale threshold)"
    "Fetch PSG lineup news (match in 52h, T-72h window opening)"
    "Run APS calculation for Player 7 (last calculated 18 days ago)"
  
  Immediate tasks are what the existing agent lifecycle handles.
  Goal levels 1-2 determine which immediate tasks are generated.
```

## Goal state machine

```
GOAL STATES:

PENDING: Goal is known but work has not started.
ACTIVE:  Goal is currently being pursued.
BLOCKED: Goal cannot progress without external information or human action.
ACHIEVED: Goal has been completed. May trigger new instrumental goals.
OBSOLETE: Goal is no longer relevant to the terminal goal.
FAILED:  Goal could not be achieved. Requires agent review.

TRANSITIONS:
  PENDING → ACTIVE:   Agent selects this goal for the current planning cycle
  ACTIVE → ACHIEVED:  Success criteria met (evidence from data or human confirmation)
  ACTIVE → BLOCKED:   Required information unavailable; agent cannot progress alone
  ACTIVE → FAILED:    Repeated attempts failed; agent flags for human review
  ACHIEVED → (new):   Achievement triggers creation of dependent goals
  Any → OBSOLETE:     Conditions changed; goal no longer serves terminal goal

EXAMPLE GOAL LIFECYCLE:
  Agent terminal goal: "Maximise PSG token portfolio over UCL campaign"
  
  Instrumental goal created: "Establish pre-match signal baseline for UCL QF"
    State: PENDING
    → ACTIVE (T-72h before match)
    → Triggers task: fetch squad news
    → Triggers task: load UCL competition context
    → Triggers task: check macro state
    → All tasks complete: ACHIEVED
    → Achievement creates new goal: "Monitor PSG signal through QF match day"
```

## Goal generation rules

```
WHEN DOES AN AGENT CREATE NEW INSTRUMENTAL GOALS?

TRIGGER 1 — TERMINAL GOAL DECOMPOSITION:
  At initialisation, the agent decomposes its terminal goal into initial
  instrumental goals. A portfolio monitor terminal goal generates:
    - "Establish token baselines for all monitored tokens"
    - "Map upcoming match schedule for monitored clubs"
    - "Confirm macro state monitoring is active"
  
TRIGGER 2 — ACHIEVEMENT CREATES DEPENDENCIES:
  When an instrumental goal is achieved, the agent checks whether new
  goals are now unlocked. Example:
    "Map upcoming match schedule" → ACHIEVED
    → Creates: "Schedule T-72h analysis for Match 1 (in 18 days)"
    → Creates: "Schedule T-72h analysis for Match 2 (in 25 days)"
    → Creates: "Flag upcoming derby (Match 3, in 32 days)"

TRIGGER 3 — OBSERVED SIGNAL CREATES GOAL:
  When the agent observes a signal that serves the terminal goal,
  it creates an instrumental goal to investigate it. Example:
    Portfolio monitor observes HAS spike for $BAR (+18 points)
    → Creates: "Investigate HAS spike source (KOL event, match result, or on-chain)"
    → Creates: "Assess whether HAS spike changes LTUI projection"
    
TRIGGER 4 — CALENDAR EVENT HORIZON:
  At regular intervals (weekly), the agent scans the calendar forward
  and creates instrumental goals for events within the planning horizon.
  Example: 30-day scan creates goals for all UCL matches in next 30 days.

TRIGGER 5 — FAILURE CREATES DIAGNOSTIC GOAL:
  When an immediate task fails repeatedly:
    "Fetch lineup news" fails 3× → Creates: "Diagnose lineup data source failure"
    Creates parallel goal: "Use alternative lineup data source"
```

```python
# core/agent_goal_framework.py
"""
SportMind Agent Goal Framework.
Extends SportMindAgent base class with goal-setting capability.
"""
from dataclasses import dataclass, field
from enum import Enum
from datetime import datetime, timezone
from typing import Optional
import json

class GoalState(Enum):
    PENDING  = "PENDING"
    ACTIVE   = "ACTIVE"
    BLOCKED  = "BLOCKED"
    ACHIEVED = "ACHIEVED"
    OBSOLETE = "OBSOLETE"
    FAILED   = "FAILED"

class GoalLevel(Enum):
    TERMINAL     = 1  # Set by human; never changes
    INSTRUMENTAL = 2  # Set by agent; changes with conditions
    IMMEDIATE    = 3  # Tasks; handled by existing agent lifecycle

@dataclass
class Goal:
    """A single goal in the agent's goal hierarchy."""
    goal_id:          str
    level:            GoalLevel
    description:      str
    state:            GoalState = GoalState.PENDING
    parent_goal_id:   Optional[str] = None
    success_criteria: str = ""
    evidence:         str = ""      # What evidence shows this is achieved?
    created_at:       str = field(default_factory=lambda: datetime.now(timezone.utc).isoformat())
    achieved_at:      Optional[str] = None
    priority:         int = 5       # 1 = highest
    dependencies:     list = field(default_factory=list)  # goal_ids that must be ACHIEVED first
    child_goals:      list = field(default_factory=list)  # goal_ids created by this goal's achievement

    def achieve(self, evidence: str = ""):
        self.state      = GoalState.ACHIEVED
        self.evidence   = evidence
        self.achieved_at = datetime.now(timezone.utc).isoformat()

    def block(self, reason: str):
        self.state    = GoalState.BLOCKED
        self.evidence = reason

    def is_actionable(self) -> bool:
        """Can this goal be worked on right now?"""
        return (self.state == GoalState.PENDING and
                all(dep in [] for dep in self.dependencies))


class GoalDirectedAgent:
    """
    Extends SportMindAgent with goal-directed behaviour.
    
    The agent maintains a goal hierarchy and uses it to determine
    what to work on next, rather than following a fixed schedule.
    """

    def __init__(self, terminal_goal: str, agent_id: str):
        self.agent_id      = agent_id
        self.goal_registry = {}  # {goal_id: Goal}
        self.goal_counter  = 0
        
        # Create the terminal goal
        self._add_goal(
            level       = GoalLevel.TERMINAL,
            description = terminal_goal,
            success_criteria = "Terminal goal by definition",
            priority    = 1
        )
        
    # ── Goal management ───────────────────────────────────────────────────────

    def _add_goal(self, level: GoalLevel, description: str,
                  success_criteria: str = "", parent_id: str = None,
                  priority: int = 5, dependencies: list = None) -> str:
        """Add a new goal to the registry."""
        self.goal_counter += 1
        goal_id = f"{self.agent_id}-G{self.goal_counter:04d}"
        
        goal = Goal(
            goal_id          = goal_id,
            level            = level,
            description      = description,
            success_criteria = success_criteria,
            parent_goal_id   = parent_id,
            priority         = priority,
            dependencies     = dependencies or []
        )
        self.goal_registry[goal_id] = goal
        
        # Link to parent
        if parent_id and parent_id in self.goal_registry:
            self.goal_registry[parent_id].child_goals.append(goal_id)
        
        return goal_id

    def achieve_goal(self, goal_id: str, evidence: str = "",
                     trigger_dependent_goals: list = None):
        """Mark a goal as achieved and optionally create dependent goals."""
        if goal_id not in self.goal_registry:
            return
        
        goal = self.goal_registry[goal_id]
        goal.achieve(evidence)
        
        # Create dependent goals unlocked by this achievement
        if trigger_dependent_goals:
            for dep_goal in trigger_dependent_goals:
                new_id = self._add_goal(
                    level            = GoalLevel.INSTRUMENTAL,
                    description      = dep_goal["description"],
                    success_criteria = dep_goal.get("success_criteria", ""),
                    parent_id        = goal_id,
                    priority         = dep_goal.get("priority", 5)
                )
                goal.child_goals.append(new_id)

    def get_next_goal(self) -> Optional[Goal]:
        """
        Determine the highest-priority actionable instrumental goal.
        This is the agent's planning function — it decides what to work on next.
        """
        actionable = [
            g for g in self.goal_registry.values()
            if g.level == GoalLevel.INSTRUMENTAL
            and g.state == GoalState.PENDING
            and g.is_actionable()
        ]
        
        if not actionable:
            return None
        
        # Highest priority first; earliest created as tiebreaker
        return sorted(actionable, key=lambda g: (g.priority, g.created_at))[0]

    # ── Planning cycle ────────────────────────────────────────────────────────

    def planning_cycle(self, observations: dict) -> list:
        """
        Core planning function: observe → evaluate → generate goals.
        
        Called at the start of each monitoring cycle.
        Returns list of immediate tasks to execute this cycle.
        
        observations = {
            "macro_state":     {"modifier": 1.00, "phase": "NEUTRAL"},
            "upcoming_events": [{"event_id": ..., "hours_away": 46}],
            "has_spikes":      [{"token": "PSG", "delta": 18}],
            "data_freshness":  {"macro_hours_old": 3.2, "lineup_confirmed": False}
        }
        """
        tasks = []
        
        # Check macro freshness
        macro_age = observations.get("data_freshness", {}).get("macro_hours_old", 0)
        if macro_age > 8:
            tasks.append({
                "task": "refresh_macro",
                "urgency": "HIGH",
                "reason": f"Macro state {macro_age:.1f}h old — approaching stale"
            })
        
        # Check for active instrumental goals that need immediate tasks
        active_goal = self.get_next_goal()
        if active_goal:
            active_goal.state = GoalState.ACTIVE
            tasks.extend(self._decompose_goal(active_goal, observations))
        
        # Generate new instrumental goals from observations
        new_goals = self._generate_goals_from_observations(observations)
        for goal in new_goals:
            self._add_goal(
                level            = GoalLevel.INSTRUMENTAL,
                description      = goal["description"],
                success_criteria = goal.get("success_criteria", ""),
                priority         = goal.get("priority", 5)
            )
        
        return tasks

    def _decompose_goal(self, goal: Goal, observations: dict) -> list:
        """Break an instrumental goal into immediate tasks."""
        tasks = []
        desc = goal.description.lower()
        
        if "pre-match signal" in desc or "analysis" in desc:
            tasks.append({"task": "fetch_lineup_news", "goal_id": goal.goal_id})
            tasks.append({"task": "load_competition_context", "goal_id": goal.goal_id})
            tasks.append({"task": "generate_signal", "goal_id": goal.goal_id})
        
        elif "baseline" in desc or "aps" in desc:
            tasks.append({"task": "calculate_aps", "goal_id": goal.goal_id})
            tasks.append({"task": "calculate_aels", "goal_id": goal.goal_id})
        
        elif "investigate" in desc and "has" in desc:
            tasks.append({"task": "check_kol_activity", "goal_id": goal.goal_id})
            tasks.append({"task": "check_onchain_events", "goal_id": goal.goal_id})
            tasks.append({"task": "check_match_result", "goal_id": goal.goal_id})
        
        return tasks

    def _generate_goals_from_observations(self, observations: dict) -> list:
        """Generate new instrumental goals from what the agent observes."""
        new_goals = []
        
        # Upcoming events → create pre-match goals
        for event in observations.get("upcoming_events", []):
            hours = event.get("hours_away", 999)
            event_id = event.get("event_id", "")
            
            # Check if we already have a goal for this event
            existing = [g for g in self.goal_registry.values()
                       if event_id in g.description]
            
            if not existing:
                if hours <= 72:
                    new_goals.append({
                        "description":    f"Generate pre-match signal for {event_id}",
                        "success_criteria": "Signal generated with SMS ≥ 60",
                        "priority":       2 if hours <= 48 else 4
                    })
        
        # HAS spikes → investigate
        for spike in observations.get("has_spikes", []):
            token = spike.get("token", "")
            delta = spike.get("delta", 0)
            if abs(delta) > 15:
                new_goals.append({
                    "description":    f"Investigate HAS spike for ${token} (delta: {delta:+.0f})",
                    "success_criteria": "Source of spike identified (match/KOL/on-chain)",
                    "priority":       3
                })
        
        return new_goals

    # ── Status and observability ──────────────────────────────────────────────

    def get_goal_status(self) -> dict:
        """Return goal status for sportmind_agent_status MCP tool."""
        by_state = {}
        for goal in self.goal_registry.values():
            state = goal.state.value
            by_state[state] = by_state.get(state, 0) + 1
        
        active_goals = [
            {"id": g.goal_id, "description": g.description, "priority": g.priority}
            for g in self.goal_registry.values()
            if g.state in (GoalState.ACTIVE, GoalState.PENDING)
            and g.level == GoalLevel.INSTRUMENTAL
        ]
        
        return {
            "terminal_goal":     next((g.description for g in self.goal_registry.values()
                                      if g.level == GoalLevel.TERMINAL), ""),
            "goal_counts":        by_state,
            "active_goals":       active_goals[:5],  # Top 5 priority
            "total_goals":        len(self.goal_registry),
            "goals_achieved":     by_state.get("ACHIEVED", 0),
            "goals_blocked":      by_state.get("BLOCKED", 0),
        }


# ── Example: Portfolio monitor with goal-directed behaviour ──────────────────

def build_portfolio_monitor_goals(agent: GoalDirectedAgent,
                                   tokens: list[str]) -> None:
    """
    Decompose a portfolio monitor terminal goal into initial instrumental goals.
    Called once at agent initialisation.
    """
    # Instrumental goal: establish baselines
    baseline_id = agent._add_goal(
        level="instrumental",  # type: ignore
        description="Establish token baselines for all monitored tokens",
        success_criteria=f"HAS, TVL tier, and macro modifier recorded for: {tokens}",
        priority=1
    )

    # Instrumental goal: map match schedule
    schedule_id = agent._add_goal(
        level="instrumental",  # type: ignore
        description="Map upcoming 30-day match schedule for monitored clubs",
        success_criteria="All matches in next 30 days identified and stored",
        priority=2
    )

    # Instrumental goal: confirm monitoring active
    agent._add_goal(
        level="instrumental",  # type: ignore
        description="Confirm macro state monitoring active and < 4h old",
        success_criteria="macro_state.json last_updated < 4h ago",
        priority=1
    )
```

## Goal-directed vs plan-directed agents

```
PLAN-DIRECTED AGENT (existing SportMind patterns 1-6):
  - Executes a fixed schedule: check macro, scan events, generate signal
  - Does not adapt to what it observes beyond immediate signal adjustments
  - Reliable and predictable; appropriate for well-defined continuous tasks
  - Use for: portfolio monitors, pre-match chains, tournament trackers

GOAL-DIRECTED AGENT (this framework):
  - Pursues a terminal goal through dynamically generated intermediate goals
  - Adapts its plan based on observations: new events, HAS spikes, data failures
  - More flexible; appropriate for long-horizon open-ended tasks
  - Use for: athlete commercial trackers, talent scouting agents,
             club operations monitors, multi-season intelligence platforms

WHEN TO USE WHICH:
  Fixed repetitive task + well-known schedule → plan-directed
  Open-ended objective + uncertain timeline → goal-directed
  Both can operate at any autonomy level (0-4)
  Both use the same safety principles from autonomous-agent-framework.md
```
